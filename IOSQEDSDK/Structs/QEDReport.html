QEDReport

public struct QEDReport : Codable

A QEDReport represents a data file locally stored on the user device that needs to be time stamped and notarised on the blockchain

Default Storage Directories
  • qedDirectory

    Default storage directory for the QEDReport represented files in the app’s bundle.

    You can use this directory to store the files you want to represent as QEDReport‘s for notarisation.

    If you use this default directory, then you can create QEDReport instances using init(withName:extension:dateTaken:)and the local fileURL location will be assumed to be on the root level of this directory.

    Declaration

    Swift

  • pdfDirectory

    Storage directory for the downloaded PDF files in the app bundle.

    Important

    When downloading PDF certificates for a QEDReport using the QEDAPI class, they will be automatically stored on the root level of this local directory.

    Note

    You can get the exact PDF url by accessing the pdfURL property of the QEDReport instance.

Managing the Associated File Hash
  • hash

    The hash to be stored on the blockchain as proof of existence of the associated file.

    Note

    This value must not be nil when issuing a report with the QEDAPI.send(_:completion:) method

    It is recommended to use SHA256 to calculate the hash, and the convenience method withCalculatedHash() will take care of this for you.

  • ==(_:_:)

    Returns a Boolean value indicating whether two values are equal.

    Two QEDReport instances are considered equal if they have the same slug (i.e. They represent the same report on the backend).

    If one or both QEDReport‘s are missing the slug, they are considered equal if they have the same dateTaken. (i.e. The instance before reporting and the instance created from report response are considered equal)
